Windowsインポートリモート証明書

Sep 14, 2020

初めに

通常、 Windows Server には RDP 証明書がインポートされていません。接続すると、証明書の警告が表示されます。

リモート証明書をインストール

p12またはpfx形式で証明書をインストール

Windows Serverに接続し、証明書をダブルクリックして、[ローカルコンピューター]> [デフォルトのファイル名]> [パスワードとその他の設定のデフォルトを入力]> [すべての証明書を次のストレージに配置-個​​人]> [完了]を選択します。

証明書ハッシュを取得する

[Win + R] > [certlm.msc] > [証明書 - ローカルコンピュータ] > [个人] > [証明書] ，インストールした証明書をダブルクリックし、[詳細]> [指紋]を選択します。

RDPサービスを構成する

Windows Powershell を管理者モードで開き、次のコマンドを実行します。

wmic /namespace:\\root\cimv2\TerminalServices PATH Win32_TSGeneralSetting Set SSLCertificateSHA1Hash="<上記で取得した証明書の指紋>"

「属性が正常に更新されました」というプロンプトが表示されます。

接続のテスト

リモートデスクトップ接続を切断して再接続します。証明書が信頼されていないことを示すものはありません。

サーバー証明書を削除

Win + R > certlm.msc > 証明書 - ローカルコンピュータ > 个人 > 証明書 > 削除
